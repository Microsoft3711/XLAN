🚀 XLAN Deployment Guide
This guide will walk you through deploying the XLAN platform — including the Blazor frontend, Azure Functions backend, and Cosmos DB — using the Azure Portal, Azure CLI (PowerShell), and GitHub Actions CI/CD.

🧭 Option 1: Deployment via Azure Portal (Manual)
1. Create a Resource Group
Portal: https://portal.azure.com

Go to Resource groups > Create

Name: xlan-rg

Region: e.g. West Europe

2. Provision Cosmos DB (NoSQL)
Create a resource → Azure Cosmos DB for NoSQL

Account name: xlan-cosmosdb

Database name: xlan-db

Container name: participants

Partition Key: /eventId

3. Create a Static Web App
Create new → Static Web App (Standard)

Name: xlan-web

Link your GitHub repository

Set build settings:

App location: client

API location: api

Azure will provide a deployment token — copy this and add it to your repo secrets as AZURE_STATIC_WEB_APPS_API_TOKEN.

4. Deploy Automatically with GitHub Actions
Ensure your repo contains:

azure-static-web-apps.yml in .github/workflows/

Project folder structure:

/client   → Blazor WebAssembly
/api      → Azure Functions


Push your changes. GitHub Actions will handle the rest 🚀


🛠 Option 2: Manual Deployment via PowerShell (Windows)
Use this script locally to deploy your backend and database:



PowerShell



# Login
az login

# Variables
$resourceGroup = "xlan-rg"
$location = "westeurope"
$storageName = "xlanstorage2025"
$functionApp = "xlan-api-func"
$cosmosName = "xlan-cosmosdb"
$databaseName = "xlan-db"
$containerName = "participants"
$partitionKey = "/eventId"

# Create resources
az group create --name $resourceGroup --location $location
az storage account create --name $storageName --resource-group $resourceGroup --location $location --sku Standard_LRS
az functionapp create --resource-group $resourceGroup --consumption-plan-location $location --runtime dotnet --functions-version 4 --name $functionApp --storage-account $storageName
func azure functionapp publish $functionApp

# Cosmos DB
az cosmosdb create --name $cosmosName --resource-group $resourceGroup --kind GlobalDocumentDB
az cosmosdb sql database create --account-name $cosmosName --name $databaseName --resource-group $resourceGroup
az cosmosdb sql container create --account-name $cosmosName --database-name $databaseName --name $containerName --partition-key-path $partitionKey --resource-group $resourceGroup

📡 2. GET /api/participants


Csharp



[FunctionName("GetParticipants")]
public static async Task<IActionResult> Run(
    [HttpTrigger(AuthorizationLevel.Function, "get", Route = "participants")] HttpRequest req,
    ILogger log)
{
    var client = new CosmosClient(Environment.GetEnvironmentVariable("CosmosDB"));
    var container = client.GetContainer("xlan-db", "participants");

    var query = new QueryDefinition("SELECT * FROM c WHERE c.IsActive = true");
    var feed = container.GetItemQueryIterator<EventParticipant>(query);

    List<EventParticipant> results = new();
    while (feed.HasMoreResults)
        results.AddRange(await feed.ReadNextAsync());

    return new OkObjectResult(results);
}




🔄 3. POST /api/session/extend – Extend user time


Csharp



public class ExtendRequest {
    public string ParticipantId { get; set; }
    public int ExtraMinutes { get; set; }
}

[FunctionName("ExtendSession")]
public static async Task<IActionResult> Run(
    [HttpTrigger(AuthorizationLevel.Function, "post", Route = "session/extend")] HttpRequest req,
    ILogger log)
{
    var body = await new StreamReader(req.Body).ReadToEndAsync();
    var input = JsonConvert.DeserializeObject<ExtendRequest>(body);

    var client = new CosmosClient(Environment.GetEnvironmentVariable("CosmosDB"));
    var container = client.GetContainer("xlan-db", "participants");

    var response = await container.ReadItemAsync<EventParticipant>(input.ParticipantId, new PartitionKey("your-event-id"));
    var participant = response.Resource;

    participant.SessionMinutes += input.ExtraMinutes;

    await container.UpsertItemAsync(participant, new PartitionKey("your-event-id"));
    return new OkObjectResult($"Extended session by {input.ExtraMinutes} minutes.");
}

This file sits at the root of your /api folder and configures global behaviors for all your functions. Here's a clean, functional version tailored to your XLAN admin dashboard endpoints:


Json

        "maxTelemetryItemsPerSecond": 5
      }
    },
    "logLevel": {
      "default": "Information",
      "Function": "Information"
    }
  },
  "extensions": {
    "http": {
      "routePrefix": "api"
    }
  }

✅ Key Points:
"version": "2.0" is required for all Functions v3 and v4.

"routePrefix": "api" ensures routes like api/participants map cleanly.

Application Insights sampling helps you monitor events without flooding logs — great for live dashboards.

If you're using CORS (for your Blazor frontend hosted separately), configure that under your Azure App Settings or via local development proxy.

🧠 3. ApiClient.cs (Service Layer)
csharp




public class ApiClient
{
    private readonly HttpClient _http;
    public ApiClient(HttpClient http) => _http = http;

    public async Task<List<EventParticipant>> GetParticipantsAsync()
        => await _http.GetFromJsonAsync<List<EventParticipant>>("api/participants");

    public async Task<List<XboxDevice>> GetDevicesAsync()
        => await _http.GetFromJsonAsync<List<XboxDevice>>("api/devices");

    public async Task<string> ExtendSessionAsync(string id, int minutes)
    {
        var body = new { participantId = id, extraMinutes = minutes };
        var res = await _http.PostAsJsonAsync("api/session/extend", body);
        return await res.Content.ReadAsStringAsync();
    }
}

Create DashboardHubClient.cs:
Csharp



using Microsoft.AspNetCore.SignalR.Client;

public class DashboardHubClient
{
    private HubConnection _hub;

    public event Action OnChange;

    public async Task ConnectAsync()
    {
        _hub = new HubConnectionBuilder()
            .WithUrl("https://your-app-url.azurewebsites.net/api")
            .WithAutomaticReconnect()
            .Build();

        _hub.On("refresh", () => OnChange?.Invoke());

        await _hub.StartAsync();
    }
}




Then in Participants.razor:

Razor



@inject DashboardHubClient Hub

protected override async Task OnInitializedAsync()
{
    participants = await Api.GetParticipantsAsync();
    await Hub.ConnectAsync();
    Hub.OnChange += async () =>
    {
        participants = await Api.GetParticipantsAsync();
        StateHasChanged();
    };
}
